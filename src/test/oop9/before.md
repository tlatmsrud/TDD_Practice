# 1. 객체지향 생활체조 9가지 법칙
1. 한 메서드에 오직 한 단계의 들여쓰기만 한다.
2. else 키워드를 사용하지 않는다. (switch case도 포함)
3. 모든 원시값과 문자열을 포장(wrap)한다.
4. 한 줄에 점을 하나만 찍는다.
5. 줄여쓰지 않는다.
6. 모든 entity를 작게 유지한다.
7. 2개 이상의 인스턴스 변수를 가진 클래스를 쓰지 않는다.
8. 일급 컬렉션을 쓴다.
9. getter/setter/property를 쓰지 않는다.

# 2. 들어가기 전
소프트윅 엔솔로지라는 책에서 소개된 법칙으로 객체지향 설계를 함에 있어 생활체조마냥 꼭 해야하는 9가지 법칙이다.  
자바지기님께서도 코드 리팩토링 시 객체지향 생활체조 9가지 법칙 준수를 강요했다.  
직접 책을 구매해 정독하고 싶었지만, 블로그에 정리된 내용도 많고 무엇보다 직접 코드를 작성함으로써 이 법칙의 의미를 이해하고 싶었다.
